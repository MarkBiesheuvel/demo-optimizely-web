Hold & send events

It is possible to activate Optimizely without sending data back to Optimizely. This is done with the holdEvents function. Visitors will still be shown experiment variations and any conversion events are queued on the visitors device. If and only if the visitor gives consent, then events will be send back to Optimizely.

Click events

The buttons below will generate click events. The Chrome DevTools (or equivalent) shows what happens with these events. If you haven't given consent yet, these events will be stored in LocalStorage. If you have given consent, these events will be send in requests to logx.optimizely.com.
